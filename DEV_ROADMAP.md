## 🟥 Phase 1 — 시중화 전 필수 (매출·보안·생존)
> v4 리스크 ①②③. 이게 안 되면 1만 명 규모에서 무너진다.

### 1-1. AI 비용·성능 최적화 ⬜
- **문제:** AI 함수 11종 × 1만 명 일일 = 비용 폭증 + 단일 인스턴스 병목
- **작업:**
  - [ ] AI 결과 캐싱 강화 (동일 입력 재호출 방지 — 서술형/요약/처방)
  - [ ] 모델 고정 점검 (Gemini Flash / Claude Haiku — Opus 잔재 제거)
  - [ ] 플랜별 AI 사용량 상한 실효성 재점검 (무료 남용 차단)
  - [ ] 무거운 조회 캐싱 확대 (학습로그·오답)
- **완료조건:** 1인당 일일 AI 콜 수 추정치 50% 절감, 동일요청 캐시 적중
- **담당:** S.Y. / 검증 A.R.

### 1-2. Polar webhook HMAC 서명 검증 ⬜
- **문제:** [확인됨] "헤더 존재만 확인" → 위조 요청으로 무단 PRO 승격 가능
- **작업:**
  - [ ] `supabase/functions/polar-webhook/index.ts` 에 Polar 서명(HMAC) 정밀 검증
  - [ ] 검증 실패 시 401 + 로깅
- **완료조건:** 위조 페이로드 거부, 정상 결제만 plan 반영
- **담당:** S.Y.

## 🟦 Phase 2 — 품질 완성 (체감 품질)
> v4 우선순위 3. 이미 만든 기능의 마감.

### 2-1. 요약노트 5mm 인쇄 실검증 ⬜
- **문제:** [확인됨] 5mm 여백이 실제 프린터에서 잘리는지 미검증(2-4)
- **작업:** 실제 PDF 출력 → 잘림 확인 → 절충값(또는 안내) 확정
- **완료조건:** A4 인쇄 시 가장자리 안 잘림
- **담당:** K.D.

### 2-2. 요약노트 오답·빈출순 정렬 ⬜
- **문제:** [확인됨] 설계(2-1)했으나 미반영 — 자주 틀린 것 우선 안 됨
- **작업:** 오답노트 데이터로 단어/문법 정렬 후 요약 생성
- **완료조건:** 자주 틀린 항목이 요약노트 상단
- **담당:** J.H. / S.Y.

### 2-3. 서술형 채점 신뢰 보강 ⬜
- **문제:** 한 달 누적 시 거짓 오답 사례 → 신뢰 흔들림
- **작업:** 채점 결과에 "이의제기 / 재채점" 버튼 + 부분점수 근거 노출
- **완료조건:** 학생이 오판 시 재채점 요청 가능
- **담당:** J.H.

## 🟩 Phase 3 — 리텐션·성장 (이탈 방어)
> v4 리스크 ④ + 그로스. 개인 학생 D30 최저 방어.

### 3-1. 개인 학생 리텐션 장치 ⬜

- **완료조건:** 개인 학생 D30 개선
- **담당:** H.S. / N.W.

### 3-2. PRO 가치 인지 강화 ⬜

- **담당:** N.W.
