# 반반 BanBan v2 — 2,000명 사용자 시뮬레이션 & 냉정한 리뷰 분석

> **문서 성격**: 실제 코드베이스(v2, 19,229줄 / 26개 모듈)에 근거한 **구조적 시뮬레이션**.
> 페르소나·점수·후기는 합성 데이터이며, 실 사용자 검증 전 **가설 기반 의사결정 자료**로만 사용한다.
> **대상**: 학생 1,000명(5그룹) · 선생님 1,000명(5그룹) · 분석 기준 20개
> **작성**: 전원 회의용 (M.J. PM / S.Y. 백엔드 / D.Y. 프론트 / A.R. QA)
> **날짜**: 2026-06-03

## 0. 시뮬레이션 방법론

### 0.1 어떻게 돌렸나
실제 구현된 기능 트리(반반노트 라이브러리·단어/문법/본문 학습·SM-2 복습·오답노트·약점 처방전·비법노트·숙제·대시보드·클래스 랭킹·학부모 리포트·반쌤 AI 챗봇·반반노트 엑셀/인쇄)를 기준선으로 삼고, 각 페르소나가 **첫 7일(온보딩) → 4주(정착) → 8주(리텐션 판정)** 의 3구간에서 보일 행동·이탈·만족도를 행동경제학·교육공학 휴리스틱으로 추정했다.

### 0.2 점수 체계
- **SUS(System Usability Scale)**: 0–100, 68이 업계 평균
- **NPS(순추천지수)**: -100 ~ +100
- **CES(Customer Effort Score)**: 1(매우 쉬움) ~ 7(매우 어려움), 낮을수록 좋음
- **D7/D30 리텐션**: 7일/30일차 재방문율
- 후기 인용("…")은 각 그룹 다수 의견을 응축한 **합성 verbatim**

### 0.3 핵심 전제 (코드에서 확인된 사실)
1. **결제/구독 기능 미구현** — `study_secret.py`에 "추후 유료 전환 예정" 주석만 존재. 따라서 "유료회원 만족도"는 **WTP(지불의향) 추정**으로 대체.
2. **Streamlit 단일 앱** — 매 상호작용마다 전체 rerun. 모바일/저사양에서 체감 지연.
3. **콘텐츠는 선생님이 직접 입력**(`영어|한글` 텍스트) 또는 OCR 업로드. **콘텐츠 공급이 곧 생명선.**
4. **AI는 Gemini 2.5 Flash 단일 키** 공유. 동시 사용자 급증 시 레이트리밋·비용 리스크.

## 3. 20개 기준 정밀 분석

> 각 기준: **점수(10점) · 그룹별 반응 · 냉정한 후기 · 근거**

### 기준 1. 온보딩 & 첫인상 — ★★★☆☆ (6.1/10)
- **호평**: 로그인 화면 디자인 깔끔, 브랜드 정체성("반반") 명확.
- **냉정한 후기**
  - S3 준호: *"들어왔는데 뭘 먼저 눌러야 하는지 모르겠어요. 메뉴가 너무 많아요."*
  - T2 박쌤: *"가입하니 바로 빈 대시보드. '첫 교재 만들기' 같은 손잡아주는 게 없어요."*
- **근거**: 온보딩 마법사 코드는 있으나 `if False`로 **비활성화 상태**. 첫 진입 시 데이터 0 → 빈 화면 다수.
- **개선 포인트**: 역할별 3스텝 온보딩 재활성화 + "샘플 노트 1개 자동 생성".

### 기준 2. 정보구조(IB)/네비게이션 — ★★★☆☆ (6.4/10)
- 사이드바 PM Dashboard 스타일 개편으로 **위계 개선**(학습하기/학습관리/대시보드/반반노트관리).
- **냉정한 후기**
  - S2 서연: *"'학습 관리'랑 '대시보드' 차이를 모르겠어요. '오답노트'는 왜 관리에 있죠?"*
  - T1 강쌤: *"메뉴가 4섹션 17개. 신규 강사 교육에 30분 걸려요."*
- **근거**: 학생에게도 `내신문제·기출문제·약점 처방전` 등 **정보 과부하**. 학생/선생 메뉴가 충분히 분리 안 됨.
- **개선 포인트**: 학생 기본 메뉴를 **5개 이하**로 축소, 고급 기능은 "더보기".

### 기준 3. 시각 디자인 일관성 — ★★★★☆ (7.6/10)
- 글래스모피즘 + Lucide 아이콘 + 디자인시스템 v4로 **동종 최고 수준의 미려함**.
- **냉정한 후기**
  - S5 하린: *"예쁜데… 좀 '어른 SaaS' 느낌? 또래가 쓰는 앱 같진 않아요."*
  - A.R. QA: *"이모지(🎉🧠🏆)와 Lucide SVG가 혼재. 지침상 이모지 금지인데 study_review 등에 잔존."*
- **근거**: HTML 영역 이모지 잔존(복습/온보딩). 디자인 토큰은 통일됐으나 **타깃(중학생) 톤**과 거리.
- **개선 포인트**: 이모지 완전 제거 + 중학생 친화 일러스트/마스코트("반쌤") 도입.

### 기준 4. 모바일 반응형 — ★★☆☆☆ (4.3/10) 🔴
- **치명적 약점**. S2·S3·S4가 **스마트폰 주력**인데 Streamlit 사이드바·`st.columns`·iframe 미리보기가 모바일에서 깨짐.
- **냉정한 후기**
  - S2 서연: *"폰에서 사이드바 열면 화면 다 가려요. 단어 카드 버튼이 손가락보다 작아요."*
  - S4 지우 학부모: *"애가 폰으로 하는데 표가 옆으로 잘려서 안 보인대요."*
- **근거**: 반반노트 출력 iframe 620px 고정, 다단 컬럼이 모바일 1열로 무너짐. 터치 타깃 <44px.
- **개선 포인트**: **모바일 우선 재설계가 v3 최우선 과제.** 카드형 단일 컬럼, 하단 탭바, 터치 타깃 ≥48px.

### 기준 5. 학습 콘텐츠 품질 (단어/문법/본문) — ★★★★☆ (7.2/10)

- 반쌤 챗봇 FAB, 대화문 키포인트, O/X 이해도 문제 등 **기능 다양**.
- **냉정한 후기**
